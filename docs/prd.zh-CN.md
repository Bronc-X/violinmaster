# ViolinMaster MVP PRD

## 一句话目标

构建一个真实的小提琴练习工作台，让学习者能够录制或上传演奏，对照所选乐谱进行比较，并获得可理解的练习诊断和针对性特训。

## 目标用户

- 在每周课程之间自主练习的成人和青少年小提琴学习者。
- 帮孩子练琴、但无法可靠听出技术错误的家长。
- 希望了解学生家庭练习情况总结的老师。

## 核心痛点

大多数小提琴学习者无法可靠判断自己的音准、节奏、换把、弓法或揉弦是否有问题。每周一次的课程过于稀疏，难以及时发现反复练错的问题；通用音乐类 App 又不会把音频分析翻译成小提琴专属的教学语言。

## MVP 承诺

完成一次练习录音后，用户应该知道：

- 哪些乐段整体比较稳定。
- 哪些乐段需要练习。
- 主要问题是音准、节奏、换把风险、四指、弓法，还是揉弦风险。
- 下一步应该练什么，并且用类似老师的普通语言说明。

## MVP 范围

### 必须具备

1. 曲目工作区
   - 浏览一个横跨铃木、考级、练习曲、音乐会曲目和中国作品元数据的精选种子库。
   - 打开带有内置或导入 MusicXML/MIDI 的受支持曲目。
   - 显示乐谱或结构化乐句地图。

2. 音频输入
   - 浏览器麦克风录音。
   - 音频文件上传。
   - 清晰的录音状态：空闲、录音中、处理中、完成、错误。

3. 全曲分析
   - 提取音高轮廓和起音/节奏特征。
   - 将演奏与所选乐谱或 MIDI 参考进行对齐。
   - 输出乐段级诊断，而不是原始技术数字。

4. 针对性特训模式
   - 用户可以选择小节范围。
   - App 只分析该范围，或从完整录音中高亮该范围。
   - AI 可以基于近期错误推荐下一段特训范围。

5. 错误汇总统计
   - 汇总多次练习中反复出现的问题。
   - 初始分类：音偏高、音偏低、节奏抢、节奏拖、换把风险片段、四指片段、弓法风险片段、揉弦风险片段。

6. AI 教练面板
   - 用友好、具体、非技术化的语言解释诊断。
   - 将底层分析转换成练习建议。
   - 默认避免暴露音分和毫秒。

7. 本地单用户历史记录
   - 在本地保存最近练习会话。
   - MVP 不做登录。

### 应该具备

- 特训模式中的节拍器和速度选择器。
- 用户录音回放。
- 在乐谱时间线上高亮问题范围。
- 对小节进行人工技术标签标注，例如换把、四指、揉弦、弓法模式。

### 不在 MVP 中

- 用户账户系统。
- 市场、教师后台、计费或社交功能。
- 仅凭音频自动可靠识别弓向或指法。
- 训练自定义基础模型。
- 演奏中的实时纠错。实时功能可以在离线分析稳定后再探索。

## 产品模型

### 主流程：完整演奏

1. 用户选择一首曲目。
2. 用户选择全曲模式。
3. 用户在浏览器中录音或上传音频文件。
4. App 处理音高和节奏。
5. App 将演奏与乐谱对齐。
6. App 显示：
   - 整体练习总结。
   - 带问题区域的时间线。
   - 前 3 个练习优先级。
   - AI 教练解释。

### 次流程：针对性特训

1. 用户打开一首曲目。
2. 用户手动选择小节范围，或接受 AI 推荐范围。
3. 用户只录制该片段。
4. App 将这次演奏与该片段进行比较。
5. App 给出简短特训指令，并把结果存入历史记录。

### 错误统计流程

1. 用户完成多次练习。
2. App 按问题类型和曲目片段分组反复出现的问题。
3. App 显示趋势语言：
   - “你的三把位换把经常落点略低。”
   - “第 16 小节之后，快速分弓音符容易赶拍。”
   - “四指音的稳定性不如替代的空弦音。”

## 诊断语言

使用类似老师的描述：

- “这个音有一点偏高。”
- “进入早了；换弓前先等住拍点。”
- “这次换把落点偏低。建议先慢慢滑到目标音。”
- “这一句里四指有一点塌。”
- “这组连弓在结尾处被压缩了。”

默认避免低层数字输出：

- 不要把“高了 37 音分”作为主要消息展示。
- 不要把“早了 143 毫秒”作为主要消息展示。
- 原始数值只保留给未来调试/开发者视图。

## 曲目数据策略

每首曲目应包含：

- 元数据：标题、作曲家、难度区间、分类、来源体系、权利状态。
- 分析素材：MusicXML 或 MIDI。
- 可选参考音频。
- 按小节或乐句标注的技术标签。
- 乐句地图：小节范围和音乐段落。

权利感知状态：

- `bundled`：完整素材可以随 App 内置。
- `import-required`：元数据存在，用户必须导入乐谱/音频。
- `private-dataset`：只有在所有者提供素材时，才可用于私人研究环境。
- `future-license`：有价值的目录条目，但暂未启用分析。

## 初始数据对象

```ts
type Piece = {
  id: string;
  title: string;
  composer: string;
  levelBand: "beginner" | "intermediate" | "advanced" | "professional";
  category: "suzuki" | "etude" | "exam" | "concerto" | "concert-piece" | "chinese";
  rightsStatus: "bundled" | "import-required" | "private-dataset" | "future-license";
  assets: {
    musicxml?: string;
    midi?: string;
    referenceAudio?: string;
  };
  techniqueMap: TechniqueLabel[];
};
```

## 成功标准

- 用户可以选择受支持曲目，并通过浏览器录音。
- 用户可以上传音频文件并获得结果。
- 至少一首内置曲目可以完成从乐谱到诊断的端到端分析。
- UI 提供全曲分析、针对性特训、统计和 AI 教练面板。
- 分析结果以小提琴学习者能理解的方式表达，而不是音频工程师语言。
- MVP 不存在只有假流程的体验：如果分析无法运行，UI 必须说明原因。

## 产品风险

- 浏览器麦克风质量差异很大。
- 复调伴奏或嘈杂房间会降低音高跟踪质量。
- MVP 中无法仅凭音频可靠推断弓法、指法和揉弦。
- 受版权保护的乐谱和音频素材在分发前需要清晰的导入/授权策略。

## MVP 风险控制

- 从独奏小提琴、单声部练习录音开始。
- 使用乐句/小节技术标签来推断小提琴专属语境。
- 显示置信度，如果音频质量太差则要求用户重录。
- 除非使用权已清晰，否则受版权保护曲目保持为元数据或用户导入素材。
